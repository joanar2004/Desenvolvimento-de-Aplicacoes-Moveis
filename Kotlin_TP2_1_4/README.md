## 1. Introduction

This exercise is part of the practical work (TP2) for the Mobile Application Development unit. The goal is to implement a `Vec2` class representing a two-dimensional vector in Kotlin, exploring the following concepts:

- **Operator overloading** (`operator fun`) — using `+`, `-`, `*`, unary `-` and `[]` directly on custom objects.
- **`Comparable<T>` interface** — implementing `compareTo` to automatically activate the `>`, `<`, `>=`, `<=` operators, and enable use in functions like `max()` and `min()`.
- **Extension functions on primitive types** — adding `operator fun Double.times(vec: Vec2)` to support the syntax `2.0 * vector`.
- **Destructuring** — the `data class` automatically provides `component1()` and `component2()`, enabling `val (x, y) = a`.
- **Vector mathematics** — implementing `magnitude()`, `dot()`, and `normalized()` based on linear algebra fundamentals.

## 2. System Overview

The system defines a `Vec2` type that behaves like a number: it can be added, subtracted, multiplied by a scalar, negated, indexed, and compared by magnitude — all using Kotlin's native operator syntax.

The program in `Main.kt` demonstrates all implemented operations on two vectors `a = (3.0, 4.0)` and `b = (1.0, 2.0)`, and also:
- Sorts a list of vectors by magnitude using the stdlib's `max()` and `min()`.
- Destructures a vector directly into `x` and `y` variables.

### Design Decisions

**`data class` as the foundation**
Using `data class` instead of a regular `class` provides automatically generated `equals()`, `hashCode()`, and `toString()`. The `toString()` produces `Vec2(x=3.0, y=4.0)`, which is exactly the format expected in the program output. Destructuring (`val (x, y) = a`) is also provided without any extra code, through the `component1()` and `component2()` methods generated by the compiler.

**`val` for immutability**
All properties are declared with `val`, making every `Vec2` instance immutable. All operations create and return a **new** `Vec2` rather than modifying the existing one — the same behaviour as primitive numbers in Kotlin.

**`Comparable<Vec2>` for magnitude-based comparison**
Implementing the `Comparable<Vec2>` interface and overriding `compareTo` automatically activates the `>`, `<`, `>=`, `<=` operators and enables the use of `max()`, `min()`, `sorted()`, and `sortedBy()` from the stdlib on lists of vectors.

**`Double.times` extension function**
To support `2.0 * vector` (scalar on the left), an extension function on `Double` is required. The `Vec2.times(scalar)` overload only supports `vector * 2.0` (scalar on the right). The extension function solves this by reusing the logic already in `Vec2.times`.

## 4. Implementation

### `Vec2.kt`

#### Class declaration

```kotlin
data class Vec2(val x: Double, val y: Double) : Comparable<Vec2> { ... }
```

The class implements `Comparable<Vec2>`, which requires overriding `compareTo`. By implementing this interface, Kotlin automatically activates the relational comparison operators (`>`, `<`, `>=`, `<=`) for any `Vec2` instance, as well as the stdlib's sorting functions.

---

#### Operator `+` — Vector addition

```kotlin
operator fun plus(other: Vec2): Vec2 {
    return Vec2(x + other.x, y + other.y)
}
```

The `operator` keyword tells the compiler that this function corresponds to the `+` operator. When the compiler encounters `a + b`, it translates it to `a.plus(b)`. The result is always a new object — `a`'s `x` and `y` are not modified.

---

#### Operator `-` — Vector subtraction

```kotlin
operator fun minus(other: Vec2): Vec2 {
    return Vec2(x - other.x, y - other.y)
}
```

Works identically to `plus`, but subtracts the components. `a - b` is translated to `a.minus(b)`.

---

#### Operator `*` — Scalar multiplication (vector on the left)

```kotlin
operator fun times(scalar: Double): Vec2 {
    return Vec2(x * scalar, y * scalar)
}
```

Unlike `plus` and `minus`, the parameter here is a `Double`, not a `Vec2`. It multiplies each component by the scalar, scaling the vector without changing its direction. Supports the syntax `a * 2.0`.

---

#### Unary operator `-` — Negation

```kotlin
operator fun unaryMinus(): Vec2 {
    return Vec2(-x, -y)
}
```

The **unary** operator acts on a single object (it receives no `other` parameter). The expression `-a` is translated to `a.unaryMinus()`. The result reverses the vector's direction while maintaining its magnitude.

---

#### `compareTo` — Comparison by squared magnitude

```kotlin
override operator fun compareTo(other: Vec2): Int {
    val thisMag = this.x * this.x + this.y * this.y
    val otherMag = other.x * other.x + other.y * other.y

    return if (thisMag < otherMag) -1
           else if (thisMag > otherMag) 1
           else 0
}
```

The `Comparable<T>` interface requires `compareTo` to return a negative integer, zero, or a positive integer depending on whether `this` is less than, equal to, or greater than `other`. The comparison uses the **squared magnitude** (`x² + y²`) instead of the real magnitude (`√(x² + y²)`), avoiding the `sqrt` operation — mathematically equivalent for comparison purposes, since `a > b ⟺ a² > b²` for non-negative values. By overriding `compareTo`, the `>`, `<`, `>=`, `<=` operators become available automatically, along with `vectors.max()` and `vectors.min()`.

---

#### `magnitude()` — Vector length

```kotlin
fun magnitude(): Double {
    return sqrt(x * x + y * y)
}
```

Applies the **Pythagorean Theorem**: in a right triangle with legs `x` and `y`, the hypotenuse (vector length) is `√(x² + y²)`. Requires `import kotlin.math.sqrt`. For `a = (3.0, 4.0)`, the result is `√(9 + 16) = √25 = 5.0`.

---

#### `dot()` — Dot product

```kotlin
fun dot(other: Vec2): Double {
    return this.x * other.x + this.y * other.y
}
```

The dot product measures the degree of alignment between two vectors. For `a = (3.0, 4.0)` and `b = (1.0, 2.0)`: `(3×1) + (4×2) = 3 + 8 = 11.0`. A positive result indicates the vectors point in the same general direction; zero indicates they are perpendicular; negative indicates opposite directions.

---

#### `normalized()` — Unit vector

```kotlin
fun normalized(): Vec2 {
    val mag = magnitude()
    if (mag == 0.0) {
        throw IllegalStateException("Cannot normalize the zero vector (division by zero).")
    }
    return Vec2(x / mag, y / mag)
}
```

Divides each component by the magnitude, producing a vector with the same direction but a magnitude of `1.0`. Throws `IllegalStateException` for the zero vector, preventing division by zero. For `a = (3.0, 4.0)` with magnitude `5.0`, the result is `(0.6, 0.8)`.

---

#### Operator `[]` — Index access

```kotlin
operator fun get(index: Int): Double {
    return when (index) {
        0    -> x
        1    -> y
        else -> throw IndexOutOfBoundsException("Invalid index $index for Vec2. Use 0 for x or 1 for y.")
    }
}
```

The `get` operator allows accessing components with array notation: `a[0]` returns `x`, `a[1]` returns `y`. Any other index throws `IndexOutOfBoundsException` with a descriptive message. The `when` expression is used expressively as a typed switch.

---

#### Extension function — `Double.times(vec: Vec2)`

```kotlin
operator fun Double.times(vec: Vec2): Vec2 {
    return vec.times(this)
}
```

This top-level function adds the `*` operator to `Double` when the right-hand operand is a `Vec2`. Without this extension, `2.0 * a` would cause a compilation error (the native `Double` does not know how to operate with `Vec2`). The implementation reuses `Vec2.times(scalar)`, passing `this` (the `Double`) as the scalar — with no duplicated logic.

### `Main.kt`

```kotlin
val a = Vec2(3.0, 4.0)
val b = Vec2(1.0, 2.0)

println("a + b = ${a + b}")            // Vec2(x=4.0, y=6.0)
println("a - b = ${a - b}")            // Vec2(x=2.0, y=2.0)
println("a * 2.0 = ${a * 2.0}")        // Vec2(x=6.0, y=8.0)
println("-a = ${-a}")                   // Vec2(x=-3.0, y=-4.0)
println("|a| = ${a.magnitude()}")      // 5.0
println("a dot b = ${a.dot(b)}")       // 11.0
println("norm(a) = ${a.normalized()}") // Vec2(x=0.6, y=0.8)
println("a[0] = ${a[0]}")              // 3.0
println("a[1] = ${a[1]}")              // 4.0
println("a > b = ${a > b}")            // true
println("a < b = ${a < b}")            // false

val vectors = listOf(Vec2(1.0, 0.0), Vec2(3.0, 4.0), Vec2(0.0, 2.0))
println("Longest = ${vectors.max()}")   // Vec2(x=3.0, y=4.0)
println("Shortest = ${vectors.min()}")  // Vec2(x=1.0, y=0.0)

val (x, y) = a
println("Destructuring: $x, $y")       // 3.0, 4.0
```

The destructuring `val (x, y) = a` works automatically because `a` is a `data class` — the compiler generates `component1()` (returns `x`) and `component2()` (returns `y`), which are called implicitly by this syntax.

## 13. Difficulties and Lessons Learned

**Unary vs. binary operators**
The distinction between `unaryMinus()` (no parameters, acts on `this`) and `minus(other: Vec2)` (binary, receives the right-hand operand) was not immediately obvious. Understanding that the compiler selects the correct function based on the number of operands in the expression clarified how Kotlin dispatches operators.

**Scalar on the right vs. scalar on the left**
`a * 2.0` and `2.0 * a` appear mathematically equivalent, but in Kotlin they are distinct operations. `a * 2.0` calls `Vec2.times(Double)`, but `2.0 * a` would attempt to call `Double.times(Vec2)` — which does not exist in the stdlib. The extension function `operator fun Double.times(vec: Vec2)` was necessary to support this second case, and it was an important lesson in how Kotlin resolves operators.

**`compareTo` and squared magnitude**
The initial approach to `compareTo` used `magnitude()` (with `sqrt`), but it became clear this was unnecessary: comparing `a² > b²` is mathematically equivalent to comparing `a > b` for magnitudes (which are always non-negative). Avoiding `sqrt` in a function that may be called frequently (in list sorting) is a good performance practice.

**`Comparable<T>` as a "multiplier switch"**
Implementing `compareTo` does not only activate `>` and `<` — it also activates `>=`, `<=`, `max()`, `min()`, `sorted()`, `coerceIn()`, and other stdlib functions. Discovering this multiplying effect from a single implementation was one of the most useful lessons of the exercise.

**Lesson learned:** Operator overloading in Kotlin is not superficial syntactic sugar — it is a coherent system based on function naming conventions. Understanding the convention (e.g. `plus`, `minus`, `times`, `get`, `compareTo`) allows writing classes that integrate naturally with the language ecosystem.

## 14. Future Improvements

- **`Float` and `Int` support:** Add `times` overloads for `Float` and `Int`, so the user does not have to write `a * 2.0` instead of the more natural `a * 2`.
- **`/` operator (scalar division):** Implement `operator fun div(scalar: Double): Vec2` to support `a / 2.0`.
- **Angle between vectors:** Add `fun angleTo(other: Vec2): Double` using `acos(dot(other) / (magnitude() * other.magnitude()))`, returning the angle in radians.
- **Distance between two points:** Add `fun distanceTo(other: Vec2): Double = (this - other).magnitude()`, useful in geometry and game development contexts.
- **Vec3:** Generalise to a 3D vector with a `z` component, adding the cross product as an additional operation.
- **Unit tests:** Write JUnit 5 tests for each operator, covering edge cases such as normalisation of the zero vector (should throw an exception), `a[2]` (should throw `IndexOutOfBoundsException`), and comparison of vectors with equal magnitude but different directions.
- **Serialisation:** Implement `kotlinx.serialization` to convert `Vec2` to/from JSON, useful in contexts involving API communication or game data persistence.
